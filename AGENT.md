# Agent Guide

This repo is in the component-contract foundation phase.

## Active Goal

Prove a reusable, framework-neutral UI/component pseudocode layer before generating full apps.

## Active Roots

- `supercss/` - visual tokens, layout primitives, and reusable component CSS
- `supercss/supercss.css` - canonical import order for active SuperCSS
- `project-to-any/templates/frontend/components/registry.json` - framework-neutral component contract registry
- `_templates/angular/` - one runnable component expo test bench
- `_rules/` - foundation rules

## Paused

- page catalogs
- route maps
- production projects
- generated content systems
- backend/API/connectors/mock DB scaffolding
- additional framework targets
- SuperCSS docs/mixer/palette/system experiments

## Working Rule

If a UI pattern is reusable, define its contract in SuperCSS and demonstrate it in the Angular component expo.

The Angular expo may contain layout scaffolding for the test bench, but it must not become a product app, dashboard, marketing site, or route catalog.
